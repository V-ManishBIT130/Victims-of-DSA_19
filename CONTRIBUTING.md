# Contributing to Axiom 2025 - DSA Repository

Thank you for your interest in contributing to the Axiom 2025 DSA repository! We're excited to have you as part of our community.

## ü§ù How to Contribute

### Adding a New Problem

1. **Create a problem file** in the `problems/` directory
   - Use the format: `problem_name.md`
   - Include: problem statement, input/output format, constraints, and examples

2. **Problem Template**
   ```markdown
   # Problem Title
   
   ## Difficulty: [Easy/Medium/Hard]
   
   ## Problem Statement
   [Describe the problem clearly]
   
   ## Input Format
   [Describe input format]
   
   ## Output Format
   [Describe output format]
   
   ## Constraints
   - List constraints here
   
   ## Examples
   
   ### Example 1:
   Input: 
   Output: 
   Explanation: 
   
   ## Tags
   [Array, String, DP, etc.]
   ```

### Adding a Solution

1. **Create a solution file** in the `solutions/` directory
   - Use the format: `problem_name_solution.[py/java/cpp/js]`
   - Include comments explaining your approach
   - Add time and space complexity analysis

2. **Solution Template**
   ```
   /*
    * Problem: [Problem Name]
    * Approach: [Brief description]
    * Time Complexity: O(?)
    * Space Complexity: O(?)
    * Author: [Your Name]
    */
   ```

### Code Style Guidelines

#### Python
- Follow PEP 8 style guide
- Use meaningful variable names
- Add docstrings for functions

#### Java
- Follow standard Java conventions
- Use camelCase for variables and methods
- Add JavaDoc comments

#### C++
- Follow Google C++ Style Guide
- Use snake_case or camelCase consistently
- Add clear comments

#### JavaScript
- Follow ES6+ standards
- Use const/let instead of var
- Add JSDoc comments

## üìã Pull Request Process

1. **Fork the repository** and create a new branch
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes**
   - Write clear, concise code
   - Test your solution thoroughly
   - Update documentation if needed

3. **Commit your changes**
   ```bash
   git add .
   git commit -m "Add: [brief description]"
   ```

4. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

5. **Create a Pull Request**
   - Provide a clear title and description
   - Reference any related issues
   - Wait for review from maintainers
